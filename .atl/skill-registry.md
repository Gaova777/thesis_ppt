## Compact Rules (auto-inject into sub-agents)

### presentationThesis/figures
Hand-written JavaScript modules produce SVG charts. Two parallel renderers (`figures-render.js` for HTML documents, `figures-deck-adapter.js` for the slide deck) share chart function names but have DIFFERENT style strategies: render.js uses CSS classes from the host HTML; deck-adapter.js inlines styles directly in SVG attributes (`font-family`, `fill`, `stroke`) with larger font sizes (22–28 px vs 12–15 px) so slides are legible at distance.

**Rule**: When modifying a chart function (`bars_h`, `heatmap`, `quadrants`, etc.), update BOTH renderers or explicitly note the divergence. The header comment in `figures-deck-adapter.js` labels this "kept in sync manually" — treat silent divergence as a bug.

**Rule**: `figures-data.js` (`window.THESIS_FIGS`) is the ONLY source of truth for figure data. Captions, labels, `reading` guides, and raw data arrays live there. Renderers are pure view layers — no data transformations beyond coordinates.

### presentationThesis/deck
`Sustentacion Tesis.html` is the main deck. It uses the `<deck-stage>` custom element defined in `deck-stage.js`. Speaker notes live in `<script type="application/json" id="speaker-notes">` inside the HTML. Slides are hidden, not unmounted — state (videos, iframes, form inputs) is preserved across navigation.

**Rule**: Design size is 1920×1080. `<deck-stage>` auto-scales via CSS transform. Adding the `noscale` attribute disables scaling (used by the PPTX exporter). Do not hardcode viewport-relative dimensions inside slide content — use the authored 1920×1080 coordinate space.

**Rule**: Press `R` to reset, arrows/space/PgUp-PgDn/Home-End to navigate, number keys to jump. Slidechange events are dispatched on the `<deck-stage>` element and bubble out of shadow DOM.
